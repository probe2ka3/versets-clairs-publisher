3. Categories of Information Processed

The Application processes the following categories of information, all of which relate exclusively to the Operator’s own TikTok account:

3.1 TikTok account information

After the Operator signs in through TikTok Login Kit (OAuth 2.0), TikTok returns to the Application a set of identifiers and tokens strictly associated with the user.info.basic scope:

  • The TikTok open ID (an opaque user identifier specific to this Application);
  • The TikTok union ID (if provided by TikTok);
  • The TikTok display name and avatar URL;
  • A short-lived access token and a refresh token issued by TikTok.

These tokens are required to call the TikTok Content Posting API on behalf of the Operator. No follower data, no follower list, no other TikTok user’s data, and no TikTok content authored by third parties is collected.

3.2 Video and caption content

The Application uploads MP4 video files and their associated text captions to TikTok when the Operator explicitly selects a video and clicks the publish or upload button. These video files are prepared off-platform; the Application stores their public URLs (in a Cloudflare R2 bucket controlled by the Operator) and the caption text in a static manifest. The Application does not collect any video content produced by other TikTok users.

3.3 Technical information

When the Operator interacts with the Application, standard server-side technical information is processed transiently to operate the service, including: HTTP request method and path, response code, approximate timestamp, IP address (for rate-limit and abuse protection only), and User-Agent string. This information is processed at the hosting layer (GitHub Pages and Cloudflare) and is not retained by the Application beyond what those providers retain for their own operational logs.

3.4 Information we do NOT collect

  • We do not collect any TikTok user’s personal data other than the Operator’s own profile.
  • We do not scrape, mine, or index public or private TikTok content.
  • We do not collect health, financial, biometric, or sensitive personal data within the meaning of Art. 5 lit. c of the Swiss Federal Act on Data Protection (FADP) or Art. 9 of the EU GDPR.
  • We do not collect information from children under the age of 16.
  • We do not place any third-party advertising or analytics cookies.

5. How Information Is Shared

The Application is designed to minimise data sharing. Information is shared exclusively in the following situations:

  • With TikTok (ByteDance Ltd. and affiliates): when the Operator publishes a video, the MP4 file and its caption are sent to TikTok’s servers using the Content Posting API. TikTok’s use of that data is governed by TikTok’s own privacy policy and developer terms.
  • With GitHub, Inc. (hosting of the web pages): the static pages of the Application (index, privacy, terms, callback) are served by GitHub Pages. GitHub processes the technical information described in §3.3 to operate its CDN. GitHub’s privacy policy applies to that processing.
  • With Cloudflare, Inc. (worker and video CDN): the OAuth code exchange and the publishing logic run inside a Cloudflare Worker controlled by the Operator. Video files are served from a Cloudflare R2 bucket controlled by the Operator. Cloudflare’s privacy policy applies to that processing.

No information processed by the Application is sold, rented, bartered, or otherwise commercialised. The Application does not provide any data to advertising networks, data brokers, or social networks other than TikTok itself.

7. Data Retention

The Application retains information only for as long as necessary for the purposes set out in §4 of this Policy, and in accordance with the following retention rules:

  • TikTok access token: retained inside a Cloudflare Workers KV namespace for the lifetime issued by TikTok (typically 24 hours) and refreshed transparently using the refresh token.
  • TikTok refresh token: retained for up to 365 days, or until the Operator revokes the authorisation from https://www.tiktok.com/setting/manage-account/app-permissions.
  • Open ID, display name, avatar: retained for the duration of an active session in the Operator’s browser (sessionStorage) and deleted when the browser tab is closed.
  • Publication queue entries (one entry per scheduled video): retained in the Cloudflare KV namespace until the publication is completed or cancelled, then deleted within 30 days.
  • Operational logs at the hosting layer: retained according to GitHub’s and Cloudflare’s own retention practices.

Upon written request from the Operator to the contact email, all tokens and queue entries associated with the Operator’s TikTok account are deleted within 30 days, except where a longer retention period is required by a binding legal obligation.

8. Security Measures

The Application implements technical and organisational security measures appropriate to the risk presented by the processing, including:

  • All communications between the Operator’s browser, the Application, and TikTok are protected by HTTPS (TLS 1.2 or higher).
  • The TikTok client_secret is stored exclusively as an encrypted secret inside the Cloudflare Worker; it is never exposed to the browser.
  • Tokens are scoped to the minimum permissions strictly required (user.info.basic, video.publish, video.upload).
  • Access to the Cloudflare account, the GitHub repository, and the TikTok developer portal is protected by strong authentication, including two-factor authentication where supported.
  • The Application is built without any third-party analytics, tracking, or advertising script.

All sessionStorage entries are deleted when the browser tab is closed.
